# Novel Framework Analyzer — Claude Code 校準文件

## 1. 專案核心目標

這個系統把中文小說切成場景，用「局心欲變」框架進行結構化分析、標註與檢索。

目標不是摘要器或問答器，而是讓使用者能討論：
- 某場景是否符合「局心欲變」
- 某角色如何在談判中改變局勢
- 角色的心態與欲望如何在局勢推動下產生轉變
- 哪些橋段最能代表某角色的策略能力、談判能力、心態成熟度

終局問題（這些問題必須能回答）：
- 寧凡哪幾場談判最能代表他的厲害之處？
- 哪一段最完整體現「局心欲變」？
- 寧凡是怎麼在弱勢中反拿主導權的？
- 輝子在哪些場景裡輸的不是戰力，而是局與心？

## 2. 框架：局心欲變（固定名稱，不可混用）

所有命名、欄位、prompt、文件、API、資料表，統一使用「局心欲變」。
禁止使用「局欲心變」「局心欲」等近似名稱。

分析順序固定：**局 → 心 → 欲 → 變**

### 局（Situation）— 外部賽局結構
- 權力關係、資源分布
- 誰主動推動、誰被迫應對
- 風險、限制、不可見邊界
- 誰握有籌碼、誰有退路

### 心（Mind）— 認知框架
- 角色進入場景時的初始心態、信念、認知偏誤
- 盲區：角色看不見或不願看見的部分
- System 1（直覺）vs System 2（理性）
- 心態的穩定性與可被撼動程度

### 欲（Desire）— 動機結構
- 顯性欲望 vs 隱性欲望
- 真正目標（可能與口頭說法不同）
- 欲望衝突與優先序
- 欲望如何被局勢放大或壓制

### 變（Change）— 轉化結果
- 觸發事件（具體行動或對話）
- 前狀態 → 後狀態（必須有可辨識前後差）
- 轉變類型：emotion / stance / strategy / values / identity
- 轉變強度 1–5（與類型分離，不可混用）
- 是否帶來成長（抗脆弱）或退縮

## 3. 最重要分析對象：寧凡談判模式（一級任務）

把「寧凡的談判場景」視為本系統的核心價值來源。

高價值談判場景特徵：
- 談判 / 討價還價 / 條件交換
- 借勢壓人 / 以退為進
- 試探底線 / 反向設局
- 用資訊差建立優勢
- 在弱勢位置取得主導權
- 表面妥協，實質推進自己目標

分析寧凡時必須回答：
1. 他是否先看懂局，再決定說什麼
2. 他是否明白對方真正怕什麼、要什麼
3. 他是否故意保留資訊，讓對方先暴露底牌
4. 他是否把原本不利的局轉成可交換的局
5. 他是否讓對方以為有選擇，但選項已被收窄
6. 他的欲是否看起來比實際更小或更合理

`is_negotiation_scene = true` 的場景必須額外輸出 `negotiation_pattern_tags`。

## 4. 系統設計原則

### 以場景為基本單位
分析主體永遠是 **Scene**，而非角色點。
在同一場景下分析多個角色各自的局心欲變。

### 先有原文證據，再有框架判斷
每個結論必須附：
- 原文句子 / 對話片段 / 觸發事件

### 類型與強度分離
- `shift_type`：emotion / stance / strategy / values / identity / none
- `shift_intensity`：1–5（強度等級，不等於類型高低）
- 禁止把 identity 自動等於高分、emotion 自動等於低分

### 不足時明說
資訊不足 → 輸出 `insufficient_context: true`，不可硬判

---

## 5. 核心資料實體

優先維護：
1. `Scene` — 最小分析單位
2. `SceneFrameworkCard` — 局心欲變分析卡
3. `NegotiationPattern` — 談判模式標籤
4. `CharacterSceneArc` — 角色弧線（從屬於 Scene）

---

## 6. 技術定位

- **SQLite → PostgreSQL**：結構化場景卡持久化（MVP 用 SQLite）
- **RAGFlow**（Phase 3）：原文知識庫與向量檢索
- **Dify**（Phase 3）：workflow / agent 編排
- **FastAPI**：中間服務層
- **Ollama qwen2.5:7b**：本地免費 LLM（8GB RAM 可跑）
- **OpenRouter Claude Haiku/Sonnet**：高品質備用

---

## 7. 嚴格避免

- 混用框架名稱
- 以角色點為主體分析（應以 scene 為主體）
- 類型與強度混淆
- match=true 但 score 極低（語義不一致）
- 把談判分析成「口才好」（要分析局心欲變動態結構）
- evidence 只有空泛描述，無原文引用
