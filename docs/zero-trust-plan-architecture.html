Zero-Trust 專案工程與程序架構圖

本頁展示零信任決策閉環系統的工程分層與程序流，所有節點均以中文標註，對應 TDD/BDD 驗證與閉環要求。

flowchart LR

二、程序閉環流程圖

flowchart TD
  啟動[觸發(Webhook/排程/人工)] --> 同步檢查[MSS 最小同步確認]
  同步檢查 -- 不一致 --> 對帳[標記 sync_drift→對帳修復]
  對帳 --> 同步檢查
  同步檢查 -- 一致 --> 事實採集[Detect 事實採集]
  事實採集 --> 放行檢查[Policy Gate 放行檢查]
  放行檢查 -- Blocked --> 人工審查[人工審查隊列]
  人工審查 --> 結束1[結束]
  放行檢查 -- Allowed --> 執行[執行原子任務]
  執行 --> 驗證[前後置驗證]
  驗證 -- Fail --> 自動恢復[自動恢復/重試/降級]
  自動恢復 --> Closeout[Closeout 知識沉澱]
  Closeout --> RAG[RAG 入庫]
  RAG --> 決策報告[決策報告]
  決策報告 --> PDCA[PDCA 回灌]
  PDCA --> 結束2[結束]
  驗證 -- Pass --> RAG
說明： 所有節點均對應 TDD/BDD 驗證要求，閉環流程每步皆有自動化測試與知識沉澱。
